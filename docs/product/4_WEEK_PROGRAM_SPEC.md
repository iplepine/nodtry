# 4주 프로그램 사양

## 목적
사용자가 너무 긴 목표 대신 4주 동안 유지 가능한 작은 실천을 파트너와 함께 완주하게 한다.

## 1차 프로그램

1차 프로그램명은 `4주 스터디 책임 스프린트`로 둔다.

대상은 성인 학습자다. 사용자는 영어, 자격증, 코딩, 독서, 글쓰기 중 하나의 작은 학습 행동을 고르고 파트너와 28일 동안 확인한다.

## 기본 구조
- 기간: 28일
- 구성: 1개 핵심 행동과 선택 설명
- 빈도: 기본 주 3일, 선택 주 5일
- 알림: 기본 저녁 9시, 선택 오전/점심/저녁 프리셋
- 파트너 역할: 승인, 확인, 응원, 똑똑, 조율
- 정산: 4주 종료 시 완료 일수, 파트너 피드백 수, 다음 플랜 제안 표시

## 주차별 경험
- 1주차: 연결, 첫 플랜 생성, 첫 3회 실천 달성
- 2주차: 실천 리듬 형성, 파트너 확인 강화
- 3주차: 지연 시 똑똑과 조율로 이탈 방지
- 4주차: 완료율 확인, 약속 정산, 다음 4주 플랜 제안

## 기본 플랜 템플릿

| 템플릿 | 기본 행동 | 기본 빈도 | 기본 알림 | 설명 |
|---|---|---:|---|---|
| 영어 | 영어 문장 10개 소리내어 읽기 | 주 5일 | 저녁 9시 | 부담 없는 매일 노출 |
| 자격증 | 기출 10문제 풀기 | 주 5일 | 저녁 9시 | 측정 가능한 반복 |
| 코딩 | 30분 코딩 공부 또는 문제 1개 | 주 3일 | 저녁 9시 | 과도한 목표 방지 |
| 독서 | 10쪽 읽고 한 줄 기록 | 주 3일 | 저녁 9시 | 인증 부담이 낮음 |
| 글쓰기 | 15분 쓰기 또는 300자 작성 | 주 3일 | 저녁 9시 | 결과물이 남는 루틴 |

구현 상태:

- 플랜 생성 첫 단계에서 위 5개 템플릿을 선택할 수 있다.
- 템플릿 선택 시 행동, 설명, 요일, 알림이 자동 입력된다.
- 새 플랜 저장 기간은 28일로 고정한다.

## 카드 노출
- 오늘 할 일이 있으면 지금 탭의 내 카드에 우선 표시한다.
- 파트너에게 확인할 일이 있으면 파트너 카드에 표시한다.
- 플랜 없음, 오늘 일정 없음, 전체 완료, 지연 상태를 명확히 구분한다.

## 완료 기준
- 플랜 종료일이 지나면 자동 완료한다.
- 완료 일수와 약속 조건을 기준으로 결과를 계산한다.
- 결과는 양측에게 정산 카드로 노출한다.
- 파일럿에서는 28일 중 12일 이상 실천을 의미 있는 완료로 본다.
- 18일 이상 실천하고 파트너 피드백을 6회 이상 받으면 강한 성공 사례 후보로 본다.

## 파일럿에서 필요한 최소 제품 흐름

1. 사용자가 스터디 템플릿 중 하나를 선택한다.
2. 기본 빈도와 알림이 자동으로 채워진다.
3. 사용자가 파트너 초대 코드를 공유한다.
4. 파트너가 플랜을 승인하거나 문구를 조율한다.
5. 사용자가 오늘 실천을 완료한다.
6. 파트너가 확인, 응원, 똑똑 중 하나를 보낸다.
7. 4주 종료 시 결과 카드가 뜬다.
8. 사용자가 다음 4주 플랜을 만들거나 종료 사유를 남긴다.

현재 구현 상태:

- 1~2번은 지원한다.
- 3~6번은 기존 앱 흐름으로 지원하되, 첫 주 압박 가시성 강화가 필요하다.
- 7~8번은 파일럿용 임시 정산 slice가 필요하다.

## 운영 메모
- 4주 프로그램은 파일럿 기본값이며, 이번 사이클에서는 더 짧거나 긴 기간을 권장하지 않는다.
- 목표는 성과 경쟁이 아니라 지속 가능한 반복 행동이다.
- 타이머, 랭킹, 공개 피드는 만들지 않는다. 경쟁이 아니라 파트너 확인이 가설이다.
