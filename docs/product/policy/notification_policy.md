# 알림(Notification) 정책

## 알림 유형

### 1. 원격 알림 (FCM)
- Firebase Cloud Messaging 기반 푸시 알림
- 이벤트 트리거: 플랜 제안, 파트너 실천 보고, 미실천 자동 전달 등

### 2. 로컬 알림 (플랜 리마인더)
- 플랜 약속 시간에 맞춘 예약 알림
- 앱 재시작 후에도 유지

## 알림 시간 설정

### 프리셋 값
| 이름 | 시간 |
|------|------|
| morning (아침) | 08:00 |
| lunch (점심) | 12:00 |
| dinner (저녁) | 20:00 |
| study sprint default | 21:00 |
| bedtime (취침 전) | 23:00 |

### 커스텀
- 사용자가 원하는 시간/분 직접 선택

### 없음 (none)
- 알림 비활성화

## 알림 스케줄링 정책

### 스케줄 생성
- 플랜 생성 또는 알림 시간 변경 시 트리거
- 선택된 각 요일의 다음 발생 시점을 계산하여 스케줄링
- `skipToday` 옵션: 당일 스케줄링 방지
- 주간 반복 설정

### 알림 ID
- 플랜 ID 해시 + 요일 번호 기반 (오버플로 방지)

### 알림 내용
- 제목: 실천 내용 (예: "30분 운동하기")
- 본문: "오늘 약속, 같이 이어갈까요?"
- 플랜 리마인더와 `poke` 푸시는 `했어`, `오늘은 패스`, `10분 후 다시 묻기` 액션을 제공한다.
- `했어` 액션 또는 플랜 리마인더 탭은 앱을 열고 해당 플랜의 메모 입력창으로 바로 이어진다.
- `오늘은 패스` 액션은 앱을 열고 확인 후 해당 플랜을 오늘 건너뜀으로 처리한다.
- `10분 후 다시 묻기` 액션은 같은 내용을 10분 뒤 1회 재알림한다.

## 미실천 자동 전달

약속 시간이 지났는데도 실천자가 오늘 행동을 처리하지 않으면, 실패가 혼자 묻히지 않도록 파트너에게 자동 전달한다.

### 조건
- 플랜 상태가 `active`
- 오늘 요일에 해당하는 항목이 있음
- 항목에 알림 시간이 설정되어 있음 (`type != none`)
- 예정 시간 + 30분이 지남
- 오늘 `completedDates`, `skippedDates`, `restedDates`, `rescuedDates` 중 어떤 처리도 없음
- `lastMissedNotifiedAt`이 오늘 날짜가 아님

### 결과
- 파트너에게 `action_missed` 타입 푸시를 보낸다.
- 플랜에 `lastMissedNotifiedAt`, `lastMissedItemTitle`, `lastUpdatedBy: system_missed_action`을 기록한다.
- 같은 플랜의 미실천 푸시는 하루 1회만 보낸다.
- 이 업데이트 자체는 일반 플랜 변경 알림으로 재전파하지 않는다.
- 앱은 `lastMissedNotifiedAt`을 읽어 Now 탭에서 실천자에게 "똑똑! 오늘 약속이 파트너에게 놓친 약속으로 남았어요.", 파트너에게 "놓친 약속이 떴어요" 상태를 보여준다.

## 알림 수명주기

| 이벤트 | 동작 |
|--------|------|
| 플랜 생성 | 알림 시간 설정 시 자동 스케줄 |
| 플랜 편집 | 기존 알림 취소 → 새 알림 스케줄 |
| 알림 비활성화 | 해당 알림 취소 |
| 플랜 완료/중단 | 관련 알림 자동 취소 |
| 회원 탈퇴 | 모든 알림 취소 |

## 권한 정책

### Android 13+ (API 33+)
- `SCHEDULE_EXACT_ALARM` 권한 필요
- 첫 토글 시 권한 요청

### iOS
- Alert, Badge, Sound 권한 요청
- 첫 토글 시 권한 다이얼로그 표시

## 시간대 지원
- `timezone` 패키지 초기화
- `flutter_timezone`으로 기기 시간대 감지
- 로컬 시간대 기준 스케줄링
- DST(일광절약시간) 자동 처리

## 알림 설정 화면
- 활성 플랜 목록 + 토글 표시
- 토글 ON/OFF: 알림 활성/비활성 (`none` 타입 전환)
- 탭하여 시간 편집
- 현재 예약된 시간 표시
