# 집중 타이머(Focus Timer) 정책

## 개요
오늘의 실천 카드(`nowAction`)에서 "지금 할게!" 버튼을 통해 즉석에서 집중 타이머를 시작할 수 있다.
타이머는 **클라이언트 로컬 상태**로만 존재하고, Firestore에는 결과(`done`)만 기록된다.
파트너는 진행 중을 보지 않고, 평소처럼 완료 결과만 본다.

## 핵심 원칙
- **진행 중은 혼자만 안다.** Firestore 동기화 / 파트너 카드 / 똑똑 푸시 로직은 그대로 둔다.
- **약속의 *타입*이 아니라 *시작 모드*다.** 모든 활성 `nowAction` 카드에서 사용 가능. 약속 생성 UX는 변경하지 않는다.
- **결과는 기존 `done` 처리.** 종료 시 기존 `ActionNoteDialog`를 띄워 `recordRepository.reportCompletion(planId, note)`을 그대로 탄다. 다만 **노트 영역에 "{N}분 집중 완료"가 prefill**돼 있어 사용자는 그대로 확인하거나 코멘트를 덧붙인다.
- **알림은 인앱만.** 로컬 푸시 / 백그라운드 알림 없음. 앱이 열려있는 동안만 동작.

## 사용자 흐름

1. `nowAction` 카드의 **"지금 할게!"** 버튼 탭.
2. 시간 입력 바텀시트: 프리셋 [5 / 10 / 25분] + 직접 입력(1~120분).
3. 타이머 화면(또는 풀스크린 바텀시트)이 열림.
   - 큰 카운트다운
   - "잠시 멈춤" / "포기" 보조
4. 0에 도달하면 **자동으로 `ActionNoteDialog`** 표시.
   - 노트 입력란에 `"{N}분 집중 완료"` prefill (예: "10분 집중 완료").
   - 사용자는 그대로 "확인"하거나 메시지를 덧붙이고 확인.
5. 확인 → 기존 `reportCompletion` 호출 → `done` 히스토리 생성. 파트너에겐 평소대로 완료가 보임.

## UI 위치
- "지금 할게!"는 카드의 **"했어"와 같은 줄 또는 바로 아래의 보조 버튼**으로 배치.
- 우선순위: `했어` (Primary) → `지금 할게!` (Secondary) → 기존 `건너뛸게` / `휴식권` 옵션.

## 중도 처리

| 행동 | 결과 |
|---|---|
| **잠시 멈춤** | 카운트다운 일시 정지. 재개 시 남은 시간으로 이어감. |
| **포기** | 타이머 종료. **아무 기록도 안 남김.** 약속은 미처리 상태 유지. 사용자가 원하면 별도로 "했어" / "건너뛸게"를 직접 누른다. |
| **앱 종료 / 백그라운드** | 알림 인프라를 쓰지 않으므로 보장하지 않음. 화면 떠나면 타이머 신뢰성 없음. 앱이 다시 활성화되면 (a) 종료시각이 지났으면 prefill 다이얼로그 즉시 표시, (b) 아직 안 지났으면 카운트다운 복구. |

> 백그라운드/킬 상태 정확성이 필요해지면 그때 로컬 알림을 추가하는 별도 슬라이스를 만든다. 1단계는 인앱 전용.

## 데이터 모델 변경
- **변경 없음.** 약속 모델 / Firestore 스키마 / 파트너 화면 모두 그대로.
- 향후 데이터 보고 prefill용 `PlanItem.defaultDurationMinutes` 추가 여부 결정. 지금은 만들지 않는다.

## 구현 슬라이스

1. **Slice A — "지금 할게!" 버튼 + 시간 선택 바텀시트**
2. **Slice B — 타이머 화면 (카운트다운, 멈춤/포기)**
3. **Slice C — 종료 시 `ActionNoteDialog` 자동 호출 + prefill 노트 ("{N}분 집중 완료")**
4. **Slice D — l10n 문자열, 햅틱, 미세 UX**

각 슬라이스 끝마다 수동 검증 + `flutter analyze`.

## 문서 갱신
- [now_tab_policy.md](./now_tab_policy.md) "내 실천 액션" 표에 "지금 할게!" 행 추가 (Slice A 완료 시).

## 영향 받지 않는 영역
- 똑똑 / `action_missed` 30분 푸시 / 파트너 카드 / 정산 로직 / 알림 권한: **변경 없음**.
- 파트너에게는 평소의 `done` 보고만 도착한다.
