# 유스케이스

## UC-01 작은 실천 계획 만들기
- 사용자: 실천자
- 상황: 새로운 루틴을 시작하려 한다.
- 흐름: 행동 입력 → 요일 선택 → 알림 설정 → 설명 추가 → 저장
- 성공 기준: 플랜이 생성되고 파트너 승인 대기 또는 활성 상태가 된다.

## UC-02 파트너와 연결하기
- 사용자: 두 명의 사용자
- 상황: 서로의 실천을 응원하려 한다.
- 흐름: 초대 코드 공유 → 코드 입력 → 유효성 확인 → 양방향 관계 생성
- 성공 기준: 연결 즉시 서로가 응원하고 지지받는 상태가 된다.

## UC-03 오늘 실천 완료하기
- 사용자: 실천자
- 상황: 지금 탭에 오늘 할 일이 보인다.
- 흐름: `했어` 선택 → 메모 입력 → 히스토리 생성
- 성공 기준: 오늘 항목이 완료되고 파트너가 확인할 수 있다.

## UC-04 파트너 실천 확인하기
- 사용자: 파트너
- 상황: 상대가 실천 완료를 보고했다.
- 흐름: 지금 탭 또는 히스토리 확인 → `실천 확인` → 리액션 또는 코멘트
- 성공 기준: 기록이 확인됨으로 표시되고 상대에게 긍정 피드백이 전달된다.

## UC-05 지연된 파트너에게 똑똑 보내기
- 사용자: 파트너
- 상황: 상대가 오늘 할 일을 아직 완료하지 않았다.
- 흐름: 지금 탭에서 `똑똑! 하기` → 메시지 또는 약속 제안
- 성공 기준: 상대가 알림을 받고 다시 행동할 계기를 얻는다.

### UC-07. 놓친 약속 자동 전달
- Goal: 약속 시간이 지났는데도 처리하지 않은 행동이 혼자 묻히지 않게 한다.
- Actor: 시스템, 파트너, 실천자
- Preconditions:
  - 실천자와 파트너가 연결되어 있다.
  - 플랜이 `active` 상태다.
  - 오늘 요일에 해당하는 항목이 있고 알림 시간이 설정되어 있다.
  - 실천자가 오늘 `done`, `skipped`, `rested`, `rescued` 중 어떤 처리도 하지 않았다.
- Trigger: 예정 시간 + 유예시간이 지난 뒤 서버 스케줄러가 실행된다.

Basic Flow:
1. 시스템이 오늘 처리되지 않은 활성 플랜을 찾는다.
2. 시스템이 같은 날짜에 이미 미실천 알림을 보냈는지 확인한다.
3. 시스템이 파트너에게 “약속 시간이 지났고 아직 처리되지 않았다”는 푸시를 보낸다.
4. 파트너는 지금 탭에서 똑똑, 실천 인정, 조율 등 후속 행동을 선택한다.
5. 실천자는 완료, 휴식권, 건너뜀, 사유 전달 중 하나로 약속을 처리한다.

Alternate Flow:
- A. 실천자가 뒤늦게 완료하면 파트너에게 완료 보고가 별도로 전달된다.
- B. 실천자가 휴식권을 쓰면 오늘은 처리 완료로 간주하고 추가 미실천 알림을 보내지 않는다.
- C. 파트너에게 FCM 토큰이 없으면 푸시는 보내지 않지만 다음 앱 진입 시 상태 카드로 확인한다.

Postconditions:
- `plans.lastMissedNotifiedAt`이 오늘 날짜로 기록된다.
- 파트너에게 `action_missed` 타입 푸시가 최대 하루 1회 전달된다.
- 오늘 행동은 사용자가 직접 처리하기 전까지 미처리 상태로 남는다.

Implementation Status:
- 부분 구현

What the user must understand:
- 약속 시간이 지나도 그냥 사라지지 않는다.
- 미처리 상태는 파트너에게 전달될 수 있다.
- 완료/휴식/건너뜀 중 하나로 직접 정리해야 한다.

What the system must explain:
- 이 압박은 사용자가 파트너를 연결하고 알림 시간을 정했기 때문에 작동한다.
- 미실천 알림은 처벌이 아니라 후속 행동을 만들기 위한 신호다.

Where to reduce choice fatigue:
- 실천자에게는 `했어`, `휴식권`, `건너뜀` 3개만 우선 노출한다.
- 파트너에게는 `똑똑`, `실천 인정`, `조율 요청` 3개만 우선 노출한다.

Defaults / recommendations to show:
- 기본 유예시간: 30분
- 기본 문구: “약속 시간이 지났어요. 한 번 물어봐 주세요.”
- 같은 플랜의 미실천 푸시는 하루 1회

## UC-06 보상/벌칙 약속 정산하기
- 사용자: 연결된 두 사용자
- 상황: 플랜 기간이 끝났다.
- 흐름: 완료 일수 집계 → 보상/벌칙 조건 계산 → 결과 카드 노출
- 성공 기준: 양측이 결과를 확인하고 다음 플랜으로 이어갈 수 있다.
