# 의사결정 기록

## D-001 파트너 관계는 양방향 상호 지지로 둔다
- 날짜: 2025-03-25
- 결정: 초대 코드 연결 시 relation 2개를 생성해 즉시 상호 지지 상태로 만든다.
- 이유: Nodtry의 파트너는 관리자가 아니라 함께 실천을 돕는 관계다.
- 영향: 똑똑, 카드 생성, 파트너 상태 확인은 관계 방향에만 의존하지 않는다.

## D-002 MVP는 1:1 파트너 루프에 집중한다
- 날짜: 2026-05-02
- 결정: 그룹, 공개 피드, 결제, AI 코치는 MVP에서 제외한다.
- 이유: 핵심 가설은 파트너 확인이 실천 유지율을 높이는지 검증하는 것이다.
- 영향: 제품 문서와 출시 준비는 1:1 플랜, 확인, 알림, 정산에 맞춘다.

## D-003 초기 버티컬 후보는 운동/건강과 학습 루틴으로 둔다
- 날짜: 2026-05-02
- 결정: 초기 메시지와 성공 사례는 운동/건강, 학습 루틴에 집중한다.
- 이유: 행동이 구체적이고 4주 내 반복 성과를 보여주기 쉽다.
- 영향: 리서치와 온보딩 예시는 두 영역부터 검증한다.

## D-004 1차 파일럿 vertical은 학습/자격증/어학으로 고정한다
- 날짜: 2026-05-03
- 결정: 다음 실행 단위는 `4주 스터디 책임 스프린트`로 둔다.
- 이유: 현재 Nodtry의 1:1 파트너 확인 구조는 공부 타이머나 랭킹보다 작은 학습 약속의 반복 확인에 더 잘 맞는다.
- 영향: 이번 사이클의 모집, 템플릿, 지표, 제품 thin slice는 성인 학습자 10쌍 파일럿에 맞춘다. 운동/건강 루틴은 2차 후보로 보류한다.
