# Nodtry 문서

Nodtry는 혼자 유지하기 어려운 작은 실천을 파트너와 함께 약속하고 확인하는 상호 지지 앱이다.

## 문서 구조
- `product/`: 제품 정의, 유스케이스, MVP 범위, 4주 프로그램
- `go-to-market/`: 초기 시장 선택, 수익 모델, 유지 지표, 성공 사례, 리서치 로그
- `operations/`: 출시 준비와 운영 체크리스트
- `decisions/`: 제품/사업 의사결정 기록
- `work/`: 현재 goal, roadmap, active task
- `product/policy/`: 현재 앱 정책의 상세 원문
- `archive/`: 더 이상 정본이 아닌 과거 문서 보관

## 정본 원칙
- 짧고 최신인 문서를 정본으로 유지한다.
- 상세 동작 정책은 `product/policy/`를 참고한다.
- 변경이 필요한 경우 새 결정은 `decisions/DECISIONS.md`에 먼저 남긴다.
