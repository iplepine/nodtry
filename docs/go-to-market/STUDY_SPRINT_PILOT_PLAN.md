# 4주 스터디 책임 스프린트 실행 계획

## 결정

Nodtry의 다음 실행 단위는 `4주 스터디 책임 스프린트`다.

이번 사이클의 목표는 기능 확장이 아니라, 성인 학습자가 파트너 확인 때문에 4주 학습 루틴을 더 잘 유지하는지 검증하는 것이다.

## 파일럿 한 줄 제안

혼자 공부가 자주 끊기는 성인 학습자 10쌍이 4주 동안 작은 공부 약속을 만들고, 파트너의 확인과 응원으로 끝까지 이어간다.

## 모집 대상

1차 모집은 10쌍, 총 20명으로 제한한다.

우선 대상:

- 영어 회화, 단어, 문장 읽기처럼 매일 반복이 필요한 성인
- 자격증 기출, 강의 수강, 문제풀이를 미루는 성인
- 코딩 공부, 독서, 글쓰기처럼 혼자 지속하기 어려운 성인

제외 대상:

- 미성년자
- 치료, 복약, 중독, 정신건강 루틴
- 회사가 직원에게 강제로 쓰게 하는 성과관리 상황
- 성적 보장이나 합격 보장을 기대하는 사용자

## 모집 메시지

짧은 버전:

> 혼자 공부가 자주 끊긴다면, 4주 동안 파트너와 작은 공부 약속을 실험해 보세요. 목표는 많이 하는 것이 아니라, 끊기지 않게 이어가는 것입니다.

상세 버전:

> Nodtry는 공부 타이머나 랭킹 앱이 아닙니다. 하루에 할 작은 공부 행동을 정하고, 파트너가 확인과 응원을 보내면서 4주 동안 이어가는 책임 실험입니다. 영어, 자격증, 코딩, 독서, 글쓰기 루틴을 혼자 유지하기 어려운 성인 학습자 10쌍을 모집합니다.

## 실행 일정

| 기간 | 목표 | 해야 할 일 | 산출물 |
|---|---|---|---|
| D-7~D-1 | 모집과 준비 | 10쌍 모집, 참여 동의, 목표 후보 수집, 앱 설치 안내 | 참여자 명단, 시작 전 설문 |
| 1주차 | 연결과 첫 실천 | 파트너 연결, 첫 플랜 생성, 첫 3회 실천 유도 | D1 활성화, W1 실천율 |
| 2주차 | 리듬 형성 | 확인/응원 반복, 알림 시간 조정, 이탈자 원인 확인 | 파트너 피드백률 |
| 3주차 | 이탈 방지 | 지연 사용자에게 똑똑/조율 유도, 목표 축소 허용 | 조율 로그 |
| 4주차 | 정산과 재참여 | 완료 일수 정산, 다음 4주 의향, 유료/보증금 의향 확인 | 결과 카드, 인터뷰 메모 |
| 종료 후 3일 | 판단 | 지표 리뷰, 성공 사례 2개 작성, 다음 실험 결정 | 파일럿 리뷰 |

## 사용자 진행 규칙

- 한 사람은 한 번에 하나의 학습 행동만 선택한다.
- 기본 기간은 28일로 고정한다.
- 기본 빈도는 주 3일 또는 주 5일 중 하나만 고른다.
- 기본 알림은 저녁 9시로 제안한다.
- 목표를 크게 잡는 것보다 첫 주 3회 성공을 우선한다.
- 파트너는 공개 감시자가 아니라, 약속이 묻히지 않게 당기는 책임 담당이다.
- 실패일은 조용히 사라지지 않고 파트너에게 전달되며, 필요하면 조율 사유로 기록한다.

## 제품 thin slice

이번 파일럿에서 필요한 제품 흐름만 구현 또는 점검한다.

1. 스터디 템플릿 선택
2. 기본 빈도/알림 자동 입력
3. 파트너 초대 코드 공유
4. 파트너 승인 또는 조율
5. 오늘 실천 완료
6. 파트너 확인/응원/똑똑
7. 지연 시 조율
8. 4주 종료 정산
9. 다음 플랜 생성 또는 종료 사유 기록

명시적으로 하지 않는 것:

- 공개 랭킹
- 공부 타이머
- AI 코치
- 다인 그룹
- 결제 구현
- HealthKit 또는 외부 인증 연동

제품 화면에서는 내부 용어인 `템플릿` 대신 `공부`, `운동`, `직접 입력` 같은 큰 카테고리를 먼저 보여준다. 이번 파일럿 분석에서는 `공부` 카테고리 사용자를 기본 표본으로 보고, 다른 카테고리는 별도 메모로 남긴다.

## 운영 체크리스트

시작 전:

- [ ] 모집 메시지 확정
- [ ] 참여자 10쌍 확보
- [ ] 시작 전 질문 5개 확정
- [ ] 앱 설치와 파트너 연결 안내 작성
- [ ] 스터디 템플릿 5개 확정
- [ ] 수동 집계 시트 또는 로그 위치 확정

1주차:

- [ ] D1 파트너 연결률 확인
- [ ] 첫 플랜 생성 실패 이유 기록
- [ ] 첫 3회 실천 달성자 확인
- [ ] 알림 기본값이 맞는지 확인

2~3주차:

- [ ] 파트너 피드백 없는 쌍 확인
- [ ] 지연 사용자 조율 여부 확인
- [ ] 똑똑이 압박으로 느껴지는지 확인
- [ ] 목표가 과도하면 줄이도록 안내

4주차:

- [ ] 완료 일수 정산
- [ ] 파트너 피드백 수 정산
- [ ] 다음 4주 재참여 의향 질문
- [ ] 9,900~19,000원 지불 의향 질문
- [ ] 보증금형 참여 의향 질문
- [ ] 성공 사례 후보 2개 선정

## 질문 스크립트

시작 전:

1. 어떤 공부를 혼자 이어가기 어렵습니까?
2. 보통 몇 일째 끊깁니까?
3. 누가 확인해 주면 부담 없이 도움이 됩니까?
4. 하루 최소 행동은 어느 정도면 현실적입니까?
5. 실패했을 때 응원, 똑똑, 조율 중 무엇이 가장 덜 부담스럽습니까?

종료 후:

1. 파트너 확인이 실제로 다음 실천에 영향을 줬습니까?
2. 똑똑은 응원이었습니까, 압박이었습니까?
3. 목표 빈도와 알림 시간은 현실적이었습니까?
4. 다음 4주를 다시 하고 싶습니까?
5. 9,900~19,000원을 내고 다시 참여할 의향이 있습니까?
6. 보증금형 참여가 있으면 더 열심히 할 것 같습니까, 부담스러울 것 같습니까?

## 성공 기준

파일럿은 아래 기준 중 4개 이상을 만족하면 다음 개발 사이클로 넘어간다.

- D1 활성화 60% 이상
- W1 3회 이상 실천 40% 이상
- W1 파트너 피드백 50% 이상
- 4주 의미 완료 25% 이상
- 다음 4주 재참여 의향 30% 이상
- 유료/보증금 의향 10% 이상
- 성공 사례로 쓸 수 있는 사용자 변화 2건 이상

## 실패 시 판단

실패해도 바로 프로젝트를 버리지 않는다. 실패 원인을 아래 중 하나로 분류한다.

- 모집 문제: 대상이 너무 넓거나 동기가 약했다.
- 목표 문제: 첫 행동이 너무 컸다.
- 파트너 문제: 확인할 사람이 없거나 부담이 컸다.
- 제품 문제: 연결, 알림, 확인, 정산 흐름이 불편했다.
- 시장 문제: 도움은 되지만 돈 낼 이유가 약했다.

시장 문제가 아니라 제품/모집 문제면 한 번 더 좁혀서 재실험한다.

## 다음 개발 순서

1. 문서 기준으로 현재 앱 흐름과 빠진 thin slice를 대조한다. `Done`
2. 스터디 템플릿과 기본값을 플랜 생성 흐름에 반영한다. `Done`
3. 파트너 확인/응원/똑똑이 첫 주에 잘 보이게 한다. `Next`
4. 4주 정산 카드 또는 임시 정산 화면을 만든다.
5. 위 흐름의 테스트 또는 시나리오 harness를 추가한다.
6. 빌드 확인 후 파일럿을 시작한다.
