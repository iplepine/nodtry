# 유지 지표

## 핵심 루프
플랜 생성 → 오늘 실천 → 파트너 확인/응원 → 다음 실천 → 4주 완료 → 다음 플랜 생성

## 4주 스터디 책임 스프린트 목표

| 지표 | 성공 기준 | 실패 기준 | 해석 |
|---|---:|---:|---|
| D1 활성화 | 60% 이상 | 35% 미만 | 첫날 파트너 연결 또는 첫 플랜 생성 |
| W1 실천 | 40% 이상 | 25% 미만 | 첫 7일 내 3회 이상 실천 |
| W1 파트너 피드백 | 50% 이상 | 30% 미만 | 첫 7일 내 확인/응원/똑똑 1회 이상 |
| 4주 의미 완료 | 25% 이상 | 15% 미만 | 28일 중 12일 이상 실천 또는 정산 완료 |
| 다음 플랜 생성 의향 | 30% 이상 | 15% 미만 | 종료 후 7일 내 다음 플랜 생성 또는 명시 의향 |
| 유료/보증금 의향 | 10% 이상 | 5% 미만 | 9,900원 이상 재참여 또는 보증금 참여 의향 |

## 활성 지표
- D1 활성화: 가입 후 24시간 내 파트너 연결 또는 첫 플랜 생성
- W1 실천: 첫 7일 내 실천 기록 3회 이상
- W1 파트너 피드백: 첫 7일 내 확인, 리액션, 똑똑 중 1회 이상
- 4주 완주: 시작한 플랜의 완료 또는 의미 있는 정산

## 유지 지표
- D7 리텐션
- D30 리텐션
- 주간 실천 기록 수
- 파트너 피드백 수신 후 다음 실천 전환율
- 플랜 완료 후 7일 내 다음 플랜 생성률

## 위험 신호
- 파트너 미연결 사용자 증가
- 첫 플랜 생성 후 첫 실천 미발생
- 파트너 확인 없는 일방 기록 증가
- 똑똑 수신 후 앱 미방문
- 파트너가 부담을 느껴 확인을 중단
- 사용자가 학습량보다 앱 관리에 피로를 느낌
- 타이머/랭킹 기능 요구가 핵심 루프보다 커짐

## 목표 가설
- 첫 주 파트너 연결률 60% 이상
- 첫 주 3회 이상 실천율 40% 이상
- 플랜 완료 후 다음 플랜 생성률 30% 이상

## 측정 이벤트

파일럿에서 최소한 아래 이벤트는 수동 집계라도 남긴다.

- `pilot_joined`: 파일럿 참여자 등록
- `partner_connected`: 파트너 연결 완료
- `study_plan_created`: 스터디 플랜 생성
- `study_action_completed`: 오늘 학습 실천 완료
- `partner_feedback_sent`: 확인, 응원, 똑똑 발송
- `plan_adjusted`: 요일, 알림, 행동 조율
- `pilot_settled`: 4주 정산 확인
- `next_plan_intent_recorded`: 다음 4주 의향 기록
- `paid_intent_recorded`: 유료/보증금 의향 기록
