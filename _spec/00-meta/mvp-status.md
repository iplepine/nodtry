# MVP 구현 현황 점검 보고서

## 1. 개요
*   **점검 일자:** 2026-01-14
*   **점검 문서:** `00-meta/mvp-spec.md`
*   **전체 상태:** 핵심 기능 약 90% 구현 완료 (일부 기능 누락 확인)

## 2. 모드 및 역할 (Role & Mode)
*   **[완료]** **실행자/관리자 모드 전환**: UI 상에서 자신의 카드와 파트너의 카드를 구분하여 노출함.
*   **[완료]** **1:N 관리 구조**: 데이터 모델(`managerId`) 상 지원하며, `NowTab`에서 파트너별 카드로 분리되어 표시됨.

## 3. 실행자 기능 (Executor)
*   **[완료]** **E1. 계획 초안 작성**: `PlanCreateScreen`을 통해 요일, 빈도, 시작/종료일 설정 가능.
*   **[완료]** **E2. 행동 수행**: `NowTab`에서 오늘 해야 할 일(Primary Card) 1개 노출.
*   **[완료]** **E3. 수행 보고**: '오늘 했어' 버튼을 통해 `reportCompletion` 실행. 선택적 메모 입력 가능.

## 4. 관리자 기능 (Manager)
*   **[부분 완료]** **M1. 계획 승인 / 반려**
    *   ✅ **승인**: `NowTab`에서 '새로운 계획 제안' 카드를 통해 승인(`approvePlan`) 가능. 승인 시 `active` 상태로 전환됨.
    *   ⚠️ **반려**: 반려 기능이 명시적으로 구현되지 않음. 현재 'Pass' 기능은 단순 `reportSkip` 로그만 남기며, 계획 상태를 `rejected`로 변경하거나 초안으로 돌려보내는 로직이 없음.
*   **[완료]** **M2. 승인된 계획 수정**: (구현 여부 확인 필요하지만, 스펙상 관리자만 수정 가능해야 함) - *현재 `PlanDetail` 수정 기능이 있으나 권한 제어가 엄격한지 확인 필요.*
*   **[완료]** **M3. 수행 결과 검증**
    *   ✅ **검증**: 파트너가 보고하면 '전달받은 말이 있어요' 카드가 뜨고, 확인(`verifyPlan`) 시 완료 처리됨.
    *   ✅ **응원**: 확인 시 칭찬/코멘트 기능(`cheerPartner`) 구현됨.

## 5. UI/UX 및 기타
*   **[완료]** **조용한 실패**: 미달성 시 별도 실패 알림 없이 넘어가며, 히스토리에는 남음.
*   **[완료]** **알림 설정**: `SettingsScreen`에서 알림 제어 가능.

## 6. 누락 및 보완 필요 사항 (TODO)
1.  **계획 반려(Reject) 기능**:
    *   관리자가 계획 제안을 거절했을 때 `PlanState.rejected`로 변경하거나 삭제하는 로직 필요.
2.  **관리자 권한 제어**:
    *   이미 승인된 계획을 실행자가 수정하지 못하도록 막는 UI 제어 확인 필요.

## 7. 결론
MVP의 **"행동 -> 보고 -> 검증"** 핵심 루프는 완성되었습니다.
다만, **계획 승인 단계에서의 '반려' 프로세스**가 누락되어 있어, 잘못된 계획이 올라왔을 때 관리자가 처리할 방법이 마땅치 않습니다. 이 부분을 추가 구현하면 MVP 스펙을 충족할 것으로 보입니다.
