# Global Role Mode Spec
## Executor / Manager Mode System

---

## 1. 설계 목적

IfTogether에서 실행자와 관리자는 **고정된 역할이 아니다**.  
모든 사용자는 상황에 따라:

- 실행자가 될 수도 있고
- 관리자가 될 수도 있다

이 구조는 **관계의 유연성**과 **책임의 순환**을 보장한다.

---

## 2. 핵심 개념 정의

### User
- 앱을 사용하는 개인
- 항상 단일 정체성

### Plan
- 특정 대상에 대해 수행되는 4주 관리 계약

### Mode
- 사용자가 현재 바라보는 관점
- UI와 행동 권한을 결정한다

---

## 3. 모드 정의

### Mode A. 실행자 모드 (Executor Mode)

의미:
> "내가 수행 중인 계획을 보는 관점"

특징:
- 내가 해야 할 행동이 중심
- 보고 버튼 노출
- 판단/검증 UI 숨김

---

### Mode B. 관리자 모드 (Manager Mode)

의미:
> "내가 관리 중인 사람들의 계획을 보는 관점"

특징:
- 관리 대상 선택 가능
- 검증/승인 UI 노출
- 실행 UI 숨김

---

## 4. 모드 전환 정책

### 전환 방식
- 앱 상단 토글 UI 제공
  - `내가 하는 중`
  - `내가 관리 중`

### 전환 시 규칙
- 데이터 손실 없음
- 현재 진행 중인 행동 취소 없음
- 단순히 UI와 권한만 전환

---

## 5. 관리자 모드 – 관리 대상 구조

### 관리 대상 (Managed Subject)
- 한 명의 사용자 또는 하나의 관계 단위
- 한 명의 관리자가 여러 대상 관리 가능

---

### 관리 대상 선택 UI

#### 위치
- 관리자 홈 상단

#### 형태
- Dropdown 또는 Chip Selector

#### 표시 정보
- 이름
- 현재 상태 요약
  - 오늘 미보고 / 대기 / 완료

---

### 기본 선택 규칙
- 가장 최근에 활동한 대상
- 또는 수동으로 마지막 선택 기억

---

## 6. 행동 권한 규칙

| 행동 | 실행자 모드 | 관리자 모드 |
|---|---|---|
| 계획 작성 | ⭕ | ❌ |
| 계획 승인 | ❌ | ⭕ |
| 수행 보고 | ⭕ | ❌ |
| 수행 검증 | ❌ | ⭕ |
| 대상 변경 | ❌ | ⭕ |

---

## 7. UX 톤 가이드

- "역할 변경"이라는 표현 사용 금지
- "모드 전환" 또는 "보기 전환" 사용
- 권력/통제 뉘앙스 제거

❌ 관리자 / 실행자 전환  
⭕ 내가 하는 중 / 내가 관리 중

---

## 8. 이 구조의 한 문장 정의

> **IfTogether에서  
> 사람은 고정되지 않고,  
> 책임만 오간다.**
