# 시간 경과(Time Passed) 상태 정책

## 0. 정책 목적
이 정책은 IfTogether 앱에서 약속한 시간이 지났을 때 시스템이 어떻게 행동하고, 어떤 문구를 보여주며, 무엇을 절대 하지 말아야 하는지를 정의한다.

### 핵심 목표
- 사용자를 판단하지 않는다.
- 관계를 불편하게 만들지 않는다.
- 행동의 회복 가능성을 항상 남긴다.

## 1. 기본 철학
시간은 사실을 말할 수 있지만, 실패를 판단해서는 안 된다.
- 시스템은 "시간이 지났다"까지만 말한다.
- "안 했다 / 실패했다"는 오직 사용자만 말할 수 있다.

## 2. 상태 정의 (단순 3단계)

| 상태 | 설명 | 비고 |
| :--- | :--- | :--- |
| **예정 (Scheduled)** | 아직 약속한 시간이 되지 않음 | 기대 상태 |
| **지남 (Time Passed)** | 약속한 시간이 지났으나 '했어' 기록이 없음 | 유예/격하 상태 |
| **완료 (Completed)** | 사용자가 직접 '했어'를 누름 | 완료 상태 |

> [!IMPORTANT]
> **실패 / 미이행 / 안 함** 상태는 시스템상에 존재하지 않는다.

## 3. 노출 규칙 (지금 탭 철학)

지금 탭은 ‘가장 임박한 다음 행동’만 책임진다. 지나간 행동은 판단하지 않고, 맥락으로만 남긴다.

### 3.1 Primary Card 선택 규칙
Primary Card는 항상 **현재 시점 이후 가장 가까운 ‘미완료 예정 행동’ 1개**를 선택한다.

- 예: A(18:00)가 지나갔고 B(18:30)가 예정되어 있다면, **B가 Primary**가 된다.

### 3.2 지나간 계획 (지남 상태) 처리
- **격하:** Primary가 아닌 지나간 계획(지남 상태)은 **Secondary Card**로 이동한다.
- **UI 제한:**
    - 버튼 없음 (기록 탭에서만 활성화 가능하도록 하거나, Now 탭에서는 노출만 함)
    - 클릭/액션 없음
    - 판단 문구 없음
- **문구:** "조금 전에 예정되어 있었어요", "앞선 약속이 있었어요"
- **Time Chip:** "조금 전", "곧 전"

### 3.3 Secondary 카드 제한
- 최대 3개까지 노출하며, 자리가 부족하면 **더 오래된 지나간 계획부터 조용히 제거**한다. (기록 탭에서만 확인 가능)

## 4. 시간 기준 정의

### 4.1 기준 시각
- 사용자가 설정한 알림 시간 (`notificationTime`)
- 알림 시간이 없을 경우: 당일 23:59

### 4.2 "지남(Time Passed)" 전환 시점
- **기준 시각 + 2시간**은 유예(grace period) 개념으로 계속 유지되나, 노출 규칙에 따라 **Primary 자리는 다음 예정 계획에 양보**한다.

## 5. 문구 정책

### 5.1 실천자(나)에게 보여주는 문구
- **예정/진행 중:** "지금 할 일이 있어요", "아직 전달된 기록이 없어요"
- **지나간 계획 (Secondary):** "조금 전에 예정되어 있었어요", "앞선 약속이 있었어요"
- **❌ 금지:** "아직 안 했어요", "실패했어요", "지각했어요"

### 5.2 매니저(상대)에게 보여주는 문구
- "아직 전달된 말이 없어요", "오늘은 조용히 지나가고 있어요"
- **원칙:** 절대 "안 함"으로 보이지 않게 하며, 설명을 요구하는 상황을 만들지 않는다.

## 6. CTA(버튼) 정책
- **Primary Card:** "했어" 버튼을 유지한다.
- **Secondary (지나간 계획):** 버튼을 제거하여 '다음 행동'에 집중하게 한다.

## 7. "안 했어"에 대한 정책
- **시스템 판단 ❌:** 시스템은 절대 자동으로 "안 했다"를 기록하지 않는다.
- **사용자 선택 ⭕ (MVP 후반):** 사용자가 직접 "오늘은 쉬어갈게"를 선택할 수 있도록 하며, 이는 상대에게 "쉬어감" 정도로만 표시된다.

## 8. 알림 정책
- **❌ 금지:** "아직 안 했어요" 알림, 반복 재촉 알림, 시간 초과 경고.
- **⭕ 허용:** 최초 알림 1회, (선택) 아주 부드러운 후속 알림 1회 (예: "오늘 약속이 아직 남아 있어요").

## 9. 확인(👍) UX와의 연결
- 시간 지남 상태에서 매니저에게 아무 행동도 요구하지 않는다.
- '했어'가 눌린 후에만 매니저 확인(👍) 루프가 열린다.

## 10. 금지 규칙 요약
- ❌ 자동 실패 판정
- ❌ 실패 색상(빨강) 사용 금지
- ❌ 통계에 "미이행 횟수" 반영 금지
- ❌ 상대에게 책임이 전가되는 문구 금지

---
**한 문장 정책 선언:**
> 지금 탭은 ‘안 한 것’을 묻는 화면이 아니라, ‘다음에 무엇을 할지’를 묻는 화면이다.
