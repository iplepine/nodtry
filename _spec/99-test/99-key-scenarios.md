# 중요 UseCase 별 시나리오 (Key Scenarios)

### UC-1: [Core Loop] 계획 생성부터 첫 실천까지
1. **계획 생성**: `UsTab` -> (+) 버튼 -> "매일 저녁 8시 독서" 입력 -> 완료.
2. **상태 확인**: `NowTab` 진입. 현재 시간이 8시 근처라면 **Now Action(Type 1)** 카드 확인.
   - 시간이 아니라면 **Vague Time(점심쯤 등)** 표시 확인.
3. **실천**: "했어" 버튼 클릭.
4. **결과 확인**:
   - `NowTab`에서 카드가 사라지거나 **Today Done(Type 2-2)** 상태로 변경됨.
   - `RecordTab`에 "독서 - 완료" 기록 추가 확인.

### UC-2: [Interaction] 파트너와 함께하기
1. **연결**: `ConnectScreen` -> 초대 코드 교환 -> 연결 완료.
2. **파트너 활동 감지**:
   - [시나리오 툴 사용]: `partnerActionShare` 선택.
   - `NowTab`에 **"지민님이 독서를 완료했어요!"(Type 4)** 카드 등장 확인.
3. **반응하기**: "확인하기" 버튼 클릭.
   - 카드가 **"함께하는 중"** 상태로 바뀌거나, 스낵바 피드백("지민님에게 응원을 보냈어요") 출력.

### UC-3: [Edge Case] 바쁜 날 챙기기 (Overdue & Reconcile)
1. **상황**: 어제 "영양제 먹기"를 못 누름.
2. **진입**: `NowTab` 확인.
3. **지난 실천 카드**: **Overdue Self Action(Type 5)** "영양제, 늦게라도 챙길까요?" 카드 확인.
4. **처리**:
   - 옵션 A: "했어" -> 어제 날짜(또는 지난 시간)로 완료 처리.
   - 옵션 B: "오늘은 넘어가자" -> 스킵(Skipped) 처리.
5. **확인**: 카드가 사라지고 `relaxedDay` 또는 다음 할 일 카드가 나옴.

### UC-4: [Onboarding] 게스트 -> 정회원 전환
1. **게스트 사용**: 게스트 로그인으로 며칠간 앱 사용 및 기록 축적.
2. **가입 유도**: 설정 -> "계정 연결/저장하기" 또는 `ConnectScreen` 진입 시 로그인 유도.
3. **소셜 로그인**: 구글 로그인 수행.
4. **데이터 유지**: 게스트 시절의 기록이 사라지지 않고 구글 계정에 연동되는지 확인.
