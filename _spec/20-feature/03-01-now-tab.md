# nod-try – 지금 탭 통합 카드 & 시간 계층 스펙
## 5가지 핵심 카드 타입 정의

## 0. 문서 목적

이 문서는 nod-try 앱의 **지금 탭**에서 사용되는 카드 타입을 5가지로 명확히 정의하고, 각 카드의 노출 조건, 표시 항목, 인터랙션을 규정한다.

---

## 1. Type 1 — Now Action Card (지금 실천 카드)
**지금 가장 가까운 시일 내 실천해야 할 항목 (Executor)**

### 1.1 노출 조건
- 아직 실천하지 않음
- 현재 시점 기준 가장 임박한 1개 (Primary)

### 1.2 표시 항목
- **제목**: 계획 이름
- **상세 설명**: 구체적인 실천 내용
- **시간 칩**: 목표 시간 또는 Vague Time
- **상태**: '액션 대기중'이 명확히 보이도록 디자인

### 1.3 인터랙션
- **확장**: 카드 터치 시 액션 영역 확장
- **액션 버튼**:
  - `했어`: 즉시 기록 생성 및 완료 처리
  - `미루기` (추후 구현)
  - `오늘은 넘어가자`: 스킵 처리

### 1.4 동작
- 버튼 클릭 시 즉시 기록 생성 후 '기록 탭'으로 이동 (선택적)
- 기록 탭에서 방금 생성된 기록 강조 (TODO)

---

## 2. Type 2 — Empty-Now Fallback Cards (빈 상황 대기 카드)
**Type 1이 없을 때만 등장하는 정보/유도 카드**

### Type 2-1 — Plan Needed Card (계획 필요)
- **노출 조건**: 활성 계획 및 한 달 내 실천 예정 없음
- **메시지**: "지금은 약속이 없어요", "한 가지 약속만 정해볼까요?"
- **행동**: 계획 추가 화면(Plan Rail / CTA)으로 이동
- **성격**: 명확한 진입 유도 (정보 카드 아님 ❌)

### Type 2-2 — Today Done / Relaxed + Preview Card (오늘 완료/여유)
- **노출 조건**: 오늘 할 게 없을 때
  - **A. 오늘 완료**: "오늘 다 했어요 🙌" + 내일 미리보기
  - **B. 원래 없음**: "오늘은 여유로운 날이에요" + 다음 일정 미리보기
- **표시 내용**: 다음 일정의 날짜, 시간, 제목
- **행동**: 버튼 없음, 체크 불가 (완전 정보용)
- **성격**: 성과 강조보다는 **"흐름이 이어지고 있음"**을 보여줌

---

## 3. Type 3 — Partner Plan Share Card (파트너 계획 공유)
**상대방의 계획 생성 및 상태 공유 (Manager)**

### 3.1 하위 타입
- **3-A. 제안**: "이런 약속을 제안했어요"
- **3-B. 거절/보류**: "이번엔 이렇게 하기로 했어요 / 조금 조정 중이에요"

### 3.2 성격
- 공유 및 인지용
- 승인/거절 액션은 이 카드 내에 없거나 별도 상세 진입 유도

---

## 4. Type 4 — Partner Action Share Card (파트너 실천 피드백)
**상대방의 실천 기록 공유 + 피드백 (Response)**

### 4.1 노출 조건
- 상대방이 실천 기록을 남김
- 아직 내가 반응(확인)하지 않음

### 4.2 표시 내용
- **상대방 프로필**: 파트너의 프로필 이미지와 이름 표시(필수)
- 상대 실천 요약
- 시간
- 상태 (했어 / 넘어갔어 등)

### 4.3 액션 (피드백)
- **봤어요**: 단순 확인
- **응원해요**: 긍정적 강화
- **그냥 넘기기**: 패스

### 4.4 성격
- '관리 업무'가 아닌 **'관계 반응'** 카드

---

## 5. Type 5 — Overdue Self Action Card (지난 내 실천)
**시간이 지나버린 내 실천 (Executor - Low Pressure)**

### 5.1 노출 조건
- 실천 시간이 지남
- 아직 처리하지 않음

### 5.2 메시지 톤
- ❌ 실패, 경고 아님
- ⭕ "지금 선택해도 괜찮아요", "늦게라도 챙겨볼까요?"

### 5.3 액션
- **했어**: 뒤늦은 완료
- **오늘은 넘어가자**: 스킵

### 5.4 성격
- Type 1과 형제지만, 압박감을 제거한 부드러운 톤

---

## 6. UI Guidelines (Visual Refinements)
- **Action Button**: `ElevatedButton`의 Vertical Padding을 줄여(12px) 날렵한 느낌 유지.
- **Partner Profile**: 파트너 컨텍스트 카드(Type 4) 및 관련 Secondary Card에는 반드시 파트너 프로필 이미지를 노출하여 '누구의 소식'인지 명확히 인지시킴.
