# IfTogether – 지금 탭 카드 계층 스펙
## Primary Action + Secondary Upcoming Cards

---

## 0. 문서 목적

이 문서는 IfTogether 앱의 **지금 탭**에서
- 현재 시점에서 가장 임박한 카드들을 어떤 "우선순위 계층"으로 보여줄지
- 각 카드가 어떤 역할과 크기를 가져야 하는지를
- 시간 표현 방식과 카드 선택 규칙을
- 시간 정보를 표현하는 UI 요소(Time Chip)를
정의하는 단일 기준 문서다.

---

## 1. 지금 탭의 시간 개념 정의

### 1.1 지금 ≠ 오늘

'지금'은 날짜 개념이 아니다.

**지금이란:**
> 현재 시점에서  
> 사용자가 가장 먼저 인지해야 할  
> 가장 임박한 행동 또는 상태

### 1.2 지금 탭에 노출 가능한 시간 범위

- 오늘 남은 행동
- 오늘이 끝났을 경우, 가장 가까운 다음 행동
  - 내일
  - 모레
  - 특정 시각까지 남은 시간

### 1.3 시간 표현 방식

'지금' 탭에서는 날짜 대신 **상대적 시간 표현만 사용한다.**

**허용되는 시간 표현:**
- `D-2`, `D-1`
- `3시간 남음`, `30분 남음`
- `곧 다가와요`
- `다음 일정까지 D-2`

**금지되는 시간 표현:**
- `2025.01.03` ❌
- `1월 3일` ❌
- `월요일` ❌
- `오늘` / `내일` (단독 사용) ❌

**시간 표현 톤 가이드:**
- 압박 ❌ / 경고 ❌ / 중립 + 안내 ⭕
- 권장: "다음 행동까지 3시간 남았어요"
- 비권장: "마감까지 3시간" ❌

---

## 2. 지금 탭 카드 계층 개념

지금 탭의 카드들은 **중요도 기준으로 계층화**된다.

**핵심 질문:**
- 지금 당장 필요한 행동은 무엇인가?
- 조금 뒤에 신경 쓰면 될 건 무엇인가?

이 두 질문의 답을 **크기와 위치**로 표현한다.

---

## 3. 카드 계층 구조 (고정)

지금 탭에는 아래 두 계층의 카드만 존재한다.

```
[ Now Primary Card ] ← 크게, 1개
[ Now Secondary Cards ] ← 작게, 1~3개
[ This Week Preview ] ← 가로 스크롤 (선택)
```

---

## 4. Now Primary Card (지금 당장 카드)

### 목적
지금 이 순간 **반드시 필요한 단 하나의 행동**을 제시한다.

### 규칙
- 지금 탭에 **항상 최대 1개**
- 버튼은 이 카드에만 허용
- 사용자가 즉시 행동할 수 있어야 함

### 크기
- Height: Large
- Padding: 24
- Radius: 16

### 예시 카드 문구
- "지금 할 일이 있어요" [했어]
- "전달받은 말이 있어요" [확인하기]
- "이번 달 계획이 아직 없어요" [계획 짜기]

---

## 5. Now Secondary Cards (조금 뒤 카드)

### 목적
가까운 미래에 발생할 수 있는 **후속 맥락**을 미리 보여준다.  
행동을 요구하지 않는다.

### 규칙
- 1~3개까지 허용
- 제목, description, 시간 정보만 노출

### 크기
- Height: Small
- Padding: 12~16
- Radius: 12

### 예시 카드 문구
- "전달했어요. 확인을 기다리고 있어요"
- "확인됐어요. 고마워요"
- "지금은 잠시 쉬어도 돼요"

---

## 6. 카드 타입 배치 규칙

### Primary Card로 올 수 있는 타입
- **Action Card** (보고 필요)
- **Response Card** (확인 필요)
- **Plan Needed Card** (계획 필요)

### Secondary Card로만 올 수 있는 타입
- **Waiting Card** (확인 대기)
- **Acknowledged Card** (확인 완료)
- **Quiet Card** (정보용)

> 동일한 타입의 카드는 중복 노출하지 않는다.

---

## 7. 카드 선택 및 배치 로직

### 7.1 Primary Card 시간 선택 규칙

Primary Card는 아래 순서로 선택된다.

1. **오늘 미완료 실천 행동**
2. **오늘 확인이 필요한 관리자 행동**
3. **오늘이 모두 완료된 경우**,  
   가장 가까운 미래 행동 (D-1, D-2, 시간 단위)
4. **아무 행동도 없을 경우** Quiet 상태

### 7.2 Primary Card 타입 결정

아래 우선순위로 **1개 선택**한다.

1. **Response Card** (확인 필요)
2. **Action Card** (보고 필요)
3. **Plan Needed Card** (계획 없음)

---

### 7.3 Secondary Cards 구성

Primary Card에 포함되지 않은 상태 중,  
의미 있는 상태를 **최대 3개까지** 선택한다.

**선택 기준:**
- 가까운 미래에 상태 변화 가능성 있음
- 사용자에게 안심 또는 맥락 제공

### 7.4 Quiet 상태 재정의

아래 조건을 **모두 만족**할 때만 Quiet 상태를 표시한다.

- 미완료 실천 행동 없음
- 확인 필요 관리자 행동 없음
- 가까운 미래 행동도 없음

**Quiet 문구 예시:**
- "지금은 잠시 쉬어도 돼요"
- "당분간 신경 쓸 일은 없어요"

- **Secondary Card 전환:** Fade (150ms)

- Primary Card 2개 이상 ❌

## 10. UX 원칙

- 탭 이름은 항상 **'지금'**으로 유지
- 카드 문구에서 날짜 대신 **시간** 사용
- **'오늘'**이라는 단어는 카드 본문에서 사용하지 않는다

## 11. 시간 UI 요소 스펙 (Time Chip)

### 11.1 핵심 설계 원칙

- 시간은 메시지가 아니다
- 시간은 상태(state)다
- 상태는 **칩(chip)** 으로 표현한다

### 11.2 Time Chip 정의

**역할:**
Time Chip은 행동의 **임박도**와 사용자가 느껴야 할 **시간 감각**을 즉각적으로 전달하는 UI 요소다.

**기본 형태:**
- Shape: Pill (둥근 캡슐)
- Height: Small
- Padding: 6~10 (Horizontal)
- Radius: Full
- 텍스트: 1줄, 매우 짧음

### 11.3 Time Chip 위치 규칙

**실천자 카드 (Executor Card):**
- 카드 상단 우측 또는 제목 위
- 항상 카드 내부에 포함

예:
```
[ D-1 ]
지금 할 일이 있어요
```

**관리자 말풍선 / 프로필 카드:**
- 상대 프로필 이미지 옆 또는 상단
- 말풍선 내부 우측 상단 정렬

예:
```
[ 민지 프로필 ]  [ 3시간 남음 ]
"운동 완료했어요"
```

### 11.4 Time Chip 타입 정의

**NOW 타입:**
- 의미: 즉시 인지해야 할 상태
- 표시 문구: "지금"
- Background: Warm Coral (`#F28B82`)
- Text: `#3F3A36`

**UPCOMING 타입:**
- 의미: 가까운 미래 행동
- 표시 문구 예: "D-1", "D-2", "3시간 남음", "30분 남음"
- Background: Light Sand (Opacity 80%)
- Text: `#7A726C`

**SOON 타입 (선택):**
- 의미: 시간 계산이 애매한 임박 상태
- 표시 문구: "곧"
- Background: Light Sand (Opacity 60%)
- Text: `#7A726C`

### 11.5 Time Chip 문구 규칙

**허용:**
- `D-1`, `D-2`
- `2시간 남음`

**금지:**
- `1월 3일` ❌
- `월요일` ❌
- 날짜(yyyy.mm.dd) 사용 금지 ❌
- 요일 사용 금지 ❌

### 11.6 Time Chip 행동 규칙

- Time Chip 자체는 클릭 불가
- 정보 전달 전용
- 행동 유도는 카드/버튼의 역할이다

### 11.7 접근성 및 인지 규칙

- 색상만으로 의미를 전달하지 않는다
- 텍스트 + 색상 조합 사용
- 작은 크기에서도 읽히도록 대비 확보

### 11.8 금지 패턴

- 문구 안에 시간 정보 섞기 ❌
- 강조 아이콘(⏰) 사용 ❌
- 애니메이션 깜빡임 ❌
- 카운트다운 실시간 감소 ❌
