# Screen – 계획 상세 (Plan Detail)

## 0. 개요
계획(약속)의 상세 내용을 확인하고 관리하는 화면.
- **나의 계획:** 수정, 알림 설정, 기록 확인.
- **상대의 계획:** 응원, 찌르기(Poke), 기록 확인.

## 1. UI 구조
- **헤더 (AppBar):** 타이틀 (스크롤 시 노출), 닫기, 메뉴 (나의 계획일 경우).
- **바디 (Scrollable):**
    -   **타이틀 & 설명:** 계획의 핵심 내용.
    -   **정보:** 빈도, 알림 시간 등.
    -   **액션 버튼 (Partner Only):** 상대방에게 반응하기 (New).
    -   **실천 기록:** 히스토리 리스트.

## 2. 주요 기능

### 2.1 찌르기 (Poke) - Partner Only
상대방이 계획을 잘 실천하고 있는지 궁금하거나, 잊은 것 같을 때 가볍게 알림을 보낸다.
- **노출 위치:** 헤더 또는 바디 상단 (잘 보이는 곳).
- **트리거:** '똑똑' 버튼.
- **조건:**
    -   상대방의 계획인 경우 노출.
    -   오늘 아직 실천 완료/인증되지 않은 경우.
    -   **제한:** 하루 1회만 가능 (자정 초기화).
- **동작:**
    -   버튼 탭 시 상대방의 '오늘' 탭에 '똑똑' 카드(State F)를 띄운다.
    -   상대방에게 푸시 알림 전송.
- **상대방 반응:**
    -   상대방은 '네' 하나로 심플하게 응답.
    -   응답 시 나에게도 알림/기록이 남을 수 있음 (Optional).
- **피드백:**
    -   전송 후 토스트 메시지 ("똑똑, 문을 두드렸어요").
    -   쿨타임 적용 (연속 전송 방지).

### 2.2 응원하기 (Cheer) - Partner Only
- 기존 '응원하기' 기능과 통합 또는 병행.
- 실천 기록에 대한 반응이 아니라, 계획 자체에 대한 독려.

## 3. 데이터 모델
- **Interaction Type:** `POKE`.
- **Payload:** Sender ID, Plan ID, Message Template ID.
