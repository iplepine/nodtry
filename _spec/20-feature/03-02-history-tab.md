# Screen 03-02 – 기록 탭 (History Tab)
## Relationship History

---

## 1. 개요 (Overview)

### 1.1 목적
기록 탭은 **상대와의 약속 이행 내역을 나열하여 서로의 행동을 기억**하는 공간이다.
평가나 비판이 아닌, **'함께 해왔음'**을 확인하고 격려하는 데 목적이 있다.

### 1.2 핵심 철학
- **관계 유지 지향:** 잘함/못함의 평가가 아닌, 지속했다는 사실 자체를 강조
- **회고보다 기억:** "지난주에 왜 안 했어?"가 아니라 "지난주에도 함께 했구나"를 느끼게 함
- **긍정적 피드백:** 실패보다는 시도와 노력에 초점을 맞춘 우호적 어조 사용

---

## 2. 정보 구조 (Structure)

### 2.1 3단 토글 필터 (Toggle Filter)
상단에 유저의 실천 주체에 따른 필터를 제공하여 인지 부담을 최소화한다.
- **모두 (All):** (기본값) 관계의 전체 흐름을 시간순으로 나열.
- **내 실천 (Me):** 내가 수행한 약속들만 표시.
- **파트너의 실천 (Partner):** 파트너가 수행한 약속들만 표시.

### 2.2 '확인해줌'의 속성 처리
"내가 확인해줬다"는 사실을 독립된 기록이 아닌, **파트너 실천 카드의 상태**로 흡수한다.
- 확인은 주인공이 아니며, 실천에 덧붙는 조용한 반응으로 정의한다.

---

## 3. UI 구성 요소 (Components)

### 3.1 기록 카드 (History Card)
주체(나/파트너)에 따라 카드 구성이 달라진다.

#### [공통] 레이아웃
- **상단:** 날짜 & 실천 상태 배지
- **중단:** 행동 내용 (Title)
- **하단:** 반응 및 확인 정보 (주체별 상이)

#### [내 실천] 카드
- **상태:** 했어 / 쉬어갔어요 / 사실 했어요
- **파트너 확인:** 파트너의 확인 여부를 작게 표시 (예: "👍 파트너가 확인함" 또는 아이콘)

#### [파트너의 실천] 카드 (핵심)
- **내 확인 상태 표시:** 내가 이 실천을 확인했는지 여부를 명확히 노출한다.
    - **채팅형 정렬 (Mirror Layout):** "누가 했는가"를 직관적으로 알 수 있도록 정렬을 활용합니다.
  - **내 실천:** 우측 정렬, 너비 85%, 우하단 꼬리(BorderRadius)
  - **파트너의 실천:** 좌측 정렬, 너비 85%, 좌하단 꼬리(BorderRadius)
- **카드 밖의 반응 (External Reaction):** 따봉(봤어요) 또는 미확인 상태 문구를 카드 하단 외부에 **12px의 의도적인 여백**을 두고 배치합니다.
- **점진적 확인 UI (Progressive Disclosure):** 반응 피로도를 낮추기 위해 액션 버튼을 평상시에는 숨기고 필요할 때만 노출합니다.
  - **평상시 (Default):** 카드 하단 우측 외부, "**아직 안 봤어요**" (연한 회색 테두리 없는 텍스트)
    - 버튼처럼 보이지 않게 설계하여 정보의 정적인 상태를 나타냄.
  - **상호작용 시 (On Tap):** 영역 터치 시 3가지 선택지 가로 나열 노출
    - **봤어요 👍** / **응원할게 💜** / **그냥 넘기기**
  - **내 확인 (봤어요 👍):** 우측 하단 앵커링 (내가 보낸 반응)
  - **파트너의 확인 (파트너가 봤어요):** 좌측 하단 앵커링 (상대의 반응)
- **카드 모서리 앵커링 (Corner Anchoring):** 반응 영역을 카드의 **안쪽 하단 모서리**에 앵커링하여, 실천 기록에 덧붙여진 따뜻한 리액션임을 강조합니다.
- **카드 간 간격:** 각 실천 단위(카드+반응) 사이에는 **24px의 여백**을 두어 쾌적한 가독성을 확보합니다.
- **상태 배지 (Top Right):**
  - '확인됨'을 별도로 표시하지 않고 '했어' 혹은 '사실 했어요' 배지만 노출하여 정보 피로도를 낮춘다.

### [POLICY] 시각적 계층 및 언어 정교화

1. **주인공은 '실천', 반응은 '덤':**
   - 상단 배지는 **실천의 상태**(했어, 쉬어갔어 등)만 표시하며, '확인됐어요' 배지는 중복이므로 제거한다.
   - 메시지(말풍선) 배경은 카드 색상과 거의 동화될 만큼 연하게 처리하여 시각적 무게를 낮춘다.
2. **직관적인 언어:**
   - "확인했어요"와 같은 딱딱한 표현 대신 "봤어요"와 같은 부드럽고 일상적인 언어를 사용한다.
   - 이미 위치(좌/우)가 주체를 말해주므로 주어를 과감히 생략하거나 축약한다.

### 3.2 Empty State (기록 없음)
기록이 없는 경우, **상태 안내 + 행동 유도** 메시지를 표시한다.
- **문구 예시 (토글별 - L10n 적용):**
    - **모두:** `historyEmptyMeTitle` ("아직 두 분의 기록이 없어요") / `historyEmptyMeSubtitle` ("첫 약속을 만들어보세요!")
    - **내 실천:** `historyEmptyMeTitle` / `historyEmptyMeSubtitle`
    - **파트너의 실천:** `historyEmptyPartnerTitle` / `historyEmptyPartnerSubtitle`

## 4. 디자인 & 감정 설계 (Design & Emotion)

### 4.1 상태 표현 가이드
- **아이콘:** 원형 배경 권장. 상태를 직관적으로 인지할 수 있는 심볼 사용.
    - 확인됨: ✔ (체크)
    - 미확인/대기: ⏳ (모래시계)
    - 이슈/경고: ! (느낌표) - *단, 강한 경고색(빨강) 지양*
- **색상:**
    - 배경: Warm Off White (`#FAF8F6`), Light Sand (`#F2ECE7`) 등 안정감 있는 톤
    - 강조(성공/긍정): Warm Coral (`#F28B82`) 등 따뜻한 톤
    - 실패/경고: 무채색 또는 중립적 세컨더리 컬러 사용 (과도한 빨강 사용 금지)

### 4.2 텍스트 톤 (Voice & Tone)
- **친근함:** 딱딱한 시스템 메시지 대신 대화체 사용
- **격려:** "함께해 준 상대에게 감사", "다시 시도해 보세요" 등의 뉘앙스
- **협력:** "등록" 대신 "**연결**", "제출" 대신 "**전달**" 등의 용어 사용

---

## 5. MVP 범위 (Scope)

### 포함 (Included)
- 날짜별 타임라인 리스트 뷰
- 각 기록의 날짜, 상태(아이콘+라벨), 행동 내용 표시
- 확인자 프로필(이미지) 및 한 줄 코멘트 표시
- 기본적인 Empty State

### 제외 (Excluded)
- 통계/분석 그래프 (성공률, 주간 리포트 등)
- 사용자 간 실적 비교/랭킹
- 날짜 필터링/검색 기능 (MVP 이후 고려)
- 복잡한 필터링 로직

---

## 6. UX 상세 (Interaction)

1.  **진입:** 사용자가 하단 '기록' 탭 탭
2.  **로딩:** 스켈레톤 UI 또는 로딩 인디케이터로 짧게 대기
3.  **데이터 없음:** Empty State 표시
4.  **데이터 있음:** 최신순 날짜별 리스트 표시
5.  **스크롤:** 무한 스크롤 또는 페이징 처리 (데이터 양에 따라)
6.  **탭/클릭:** (MVP) 상세 화면 이동 없음 (카드 내 정보가 전부) 또는 카드 확장
