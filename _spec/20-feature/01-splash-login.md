# Screen 01 – 스플래시 & 로그인
## Warm Accountability Entry Point

---

## 1. 화면 목적 (Why)

이 화면의 목적은 **서비스 설명이 아니다**.  
사용자에게 다음 감정을 심어주는 것이 목적이다.

- “뭔가 관리당하는 앱은 아니구나”
- “조용하고 믿을 수 있는 느낌이다”
- “빨리 시작해도 부담 없겠다”

👉 **설명 ❌ / 분위기 ⭕**

---

## 2. 행동 정책 (Behavior Policy)

### B1. 최소 개입 원칙
- 앱은 사용자를 설득하지 않는다
- 선택지를 많이 주지 않는다
- “왜 로그인해야 하는지” 설명하지 않는다

### B2. 판단 유예
- 실행자/관리자 역할 선택은 여기서 하지 않는다
- 목표 설정, 계획, 관리 개념은 이 화면에 노출하지 않는다

> 첫 진입에서 책임/목표를 들이밀면 이탈 확률이 높아진다

---

## 3. 사용자 흐름 (User Flow)

### 3.1 스플래시
- 앱 실행
- 자동 로그인 시도
- 분기:
  - 로그인 O → 커플 연결 여부 확인 → 다음 화면
  - 로그인 X → 로그인 화면

---

### 3.2 로그인
- **Android:**
  - Google 로그인
  - 둘러보기 (게스트 모드)
- **iOS:**
  - 둘러보기 (게스트 모드) ONLY (Apple 심사 회피용)
  - *추후 Apple 로그인 추가 가능*
  - *추후 Apple 로그인 추가 가능*
- **이메일 로그인:**
  - 이메일/비밀번호 회원가입 및 로그인 지원

---

## 4. UI 구성 요소

### 4.1 스플래시 화면

#### 구성
- 로고 (중앙)
- 태그라인 (1줄, 선택)

#### 태그라인 예시
- “혼자서 지키지 못했던 계획을, 함께”
- “나를 대신해, 누군가가 관리해주는 시간”

#### 정책
- 로딩 스피너 과도한 강조 금지
- 애니메이션은 매우 느리고 부드럽게 (Fade)

---

### 4.2 로그인 화면

#### 구성
- 상단: 여백
- 중앙: 로그인 버튼
- 하단: 짧은 신뢰 메시지 + **둘러보기 버튼**

#### 버튼
- Primary:
  - “Google로 시작하기”
  - “Apple로 시작하기”
- Secondary (Text Button):
  - "로그인 없이 시작하기" (데이터는 기기에만 저장됩니다)

#### 하단 메시지 (아주 작게)
- “강요하지 않아요. 기록은 둘만 봅니다.”

## 5. 디자인 가이드

### 5.1 컬러 사용

#### 배경
- `Warm Off White #FAF8F6`

#### 버튼
- Primary Button:
  - Background: `Warm Coral #F28B82`
  - Text: `#3F3A36`

#### 텍스트
- Primary Text: `#3F3A36`
- Secondary Text: `#7A726C`

❌ 경고 색상 / ❌ 대비 강한 컬러 금지

---

### 5.2 타이포그래피 톤

- 큰 제목 없음
- 문장은 짧고 부드럽게
- 명령형 금지

❌ “로그인하세요”  
⭕ “시작해볼까요?”

---

## 6. 마이크로 카피 가이드

| 상황 | 금지 | 권장 |
|---|---|---|
| 로그인 | 필수입니다 | 시작해볼까요 |
| 권한 | 허용하세요 | 함께 쓰기 위해 필요해요 |
| 오류 | 실패 | 잠시 안됐어요 |
