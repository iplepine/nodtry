# Screen 04 – 공통 헤더
## Quiet Context Header

---

## 0. 헤더의 철학

nod-try의 헤더는 **조작을 유도하는 UI가 아니다**.  
헤더는 다음 역할만 수행한다.

- "지금 누구와 연결돼 있는지"
- "지금 어떤 기간을 살고 있는지"
- "설정은 어디 있는지"

즉, 헤더는 **안심용 맥락 표시줄**이다.

---

## 1. 헤더가 절대 하지 않는 것

- 행동을 유도하지 않는다
- 계획을 강조하지 않는다
- 성과를 보여주지 않는다
- 역할(실행자/관리자)을 드러내지 않는다

헤더는 **홈 카드보다 항상 존재감이 약해야 한다**.

---

## 2. 헤더 기본 구조

```
[ 좌측 ]        [ 중앙 ]            [ 우측 ]
 관계 정보        기간 정보            설정
```

---

## 3. 좌측 영역 – 관계 정보

### 목적
"지금 누구와 함께 쓰고 있는지"를 알려준다

### 표시 내용 (우선순위)
- 상대 이름 (기본)
  - 예: 민지
- 관계 별칭 (선택)
  - 예: 우리, 가족, 운동 메이트

### 노출 규칙
- 텍스트 1줄
- 강조 금지
- 아이콘 없음

### 인터랙션
- 클릭 불가 (MVP)
- 드롭다운 없음

---

## 4. 중앙 영역 – 기간 정보

### 목적
"지금 어떤 흐름 안에 있는지"를 알려준다

### 기본 포맷
```
4주 중 · 2주차
```

### 상태별 표시

| 상태 | 표시 |
|---|---|
| 진행 중 | 4주 중 · 2주차 |
| 계획 없음 | 아직 계획이 없어요 |
| 종료 직후 | 4주가 끝났어요 |

### 정책
- 날짜(yyyy.mm.dd) 표시 금지
- 카운트다운 금지
- D-Day 표현 금지

---

## 5. 우측 영역 – 설정 진입

### 목적
"언제든 빠져나갈 수 있다"는 안정감 제공

### 구성
- 아이콘: ⚙️ (설정)

### 인터랙션
- 탭 → 🤍 우리 탭의 설정 섹션으로 이동
- 롱프레스 없음
- 뱃지 없음

---

## 6. 헤더 디자인 가이드

### 컬러
- 텍스트: Secondary Text
- 아이콘: Secondary Text
- 배경: 투명 또는 Background

### 타이포그래피
- 크기: Body Small
- 굵기: Regular
- 대문자 사용 금지

### 레이아웃
- 높이: 낮게 (Compact)
- 그림자 없음
- 구분선 없음 (필요 시 아주 연한 Divider)

---

## 7. Today Card와의 관계

- 헤더는 정보
- Today Card는 행동의 중심

**시각적 우선순위:**
1. Today Card
2. Header
3. Bottom Tabs

---

## 8. 예외 상황 처리

### 여러 관계를 동시에 관리하는 경우
- 홈에서는 현재 선택된 관계 1개만 표시
- 관계 전환은 🤍 우리 탭에서만 가능

### 오프라인 / 로딩 중
- 헤더 영역 Skeleton 처리
- 문구 표시 ❌

---

## 9. 헤더 한 문장 정의

> 상황을 설명하지 않고,  
> 안심만 남기는 줄이다.**
