# Screen 03 – 홈 (Today Card)
## Relationship-Centered Home

---

## 0. 이 문서의 위치

이 문서는 OnMyBehalf의 **홈 화면 단일 기준 스펙**이다.

- 실행자 홈 ❌
- 관리자 홈 ❌
- 역할 토글 ❌

OnMyBehalf의 홈은 **하나의 화면, 하나의 카드**로 정의된다.

---

## 1. 화면 정의 (What)

홈 화면은 OnMyBehalf의 **유일한 중심 화면**이다.

이 화면은:
- 역할을 묻지 않는다
- 할 일을 나열하지 않는다
- 성과를 평가하지 않는다

홈은 오직 다음 질문에만 답한다.

> **"오늘, 우리 사이에 오고 가야 할 말이 있나?"**

---

## 2. 설계 목적 (Why)

이 화면의 목적은:
- 생산성 향상 ❌
- 자기관리 강화 ❌

이 화면의 목적은:
- 관계의 흐름을 끊지 않기 ⭕
- 책임을 부드럽게 이어주기 ⭕
- 아무 일 없는 날도 편안하게 만들기 ⭕

> 이 앱은 "매일 써야 하는 앱"이 아니라  
> "열었을 때 부담 없는 앱"이어야 한다.

---

## 3. 핵심 설계 원칙 (Home Principles)

### H1. 홈은 카드 한 장이다
- 홈에는 **항상 하나의 메인 카드만** 존재한다
- 추가 정보는 카드 아래에 보조 정보로만 제공한다

### H2. 홈은 "상태"를 보여준다
- 행동을 지시하지 않는다
- 오늘의 관계 상태를 전달한다

### H3. 버튼은 최대 하나
- 행동이 필요한 날만 버튼이 존재한다
- 버튼이 없는 날도 정상 상태다

---

## 4. Today Card 상태 모델

Today Card는 아래 상태 중 **하나만** 노출된다.

---

### 상태 A. 내가 해야 할 말이 있을 때 (보고 필요)

#### 노출 조건
- 오늘 수행해야 할 플랜이 존재
- 아직 보고하지 않음

#### 카드 내용
```
오늘 할 일이 있어요
[ 했어 ]
```

#### 행동
- `했어` 버튼 탭 → 수행 보고 생성
- 상태 전이 → 상태 C

#### 정책
- 이유 설명 없음
- 목표 설명 없음
- 판단 문구 없음

---

### 상태 B. 내가 들어야 할 말이 있을 때 (확인 필요)

#### 노출 조건
- 관리 중인 대상의 보고가 존재
- 아직 확인하지 않음

#### 카드 내용
```
전달받은 말이 있어요
[ 확인하기 ]
```

#### 행동
- `확인하기` 탭 → 확인/검증 화면 진입
- 확인 완료 → 상태 D 또는 E

---

### 상태 C. 전달은 했고, 확인을 기다리는 중

#### 노출 조건
- 내가 보고를 완료함
- 상대 확인 전

#### 카드 내용
```
전달했어요
확인을 기다리고 있어요
```

#### 행동
- 버튼 없음

---

### 상태 D. 오늘은 오갈 말이 없음

#### 노출 조건
- 보고 필요 없음
- 확인 필요 없음

#### 카드 내용
```
오늘은 조용한 하루예요
```

#### 행동
- 버튼 없음

---

### 상태 E. 확인이 완료됨

#### 노출 조건
- 내가 한 보고가 확인됨

#### 카드 내용
```
확인됐어요
고마워요
```

#### 행동
- 버튼 없음

---

## 5. 상태 우선순위 규칙

여러 상태가 동시에 가능한 경우,  
카드는 아래 우선순위로 하나만 선택된다.

1. **상태 B** – 확인 필요
2. **상태 A** – 보고 필요
3. **상태 C** – 확인 대기
4. **상태 E** – 확인 완료
5. **상태 D** – 조용한 하루

홈에는 항상 하나의 메시지만 존재한다.

---

## 6. 카드 하단 보조 정보 (Context Footer)

Today Card 아래에는 행동을 유도하지 않는 정보만 노출된다.

#### 예시
- `4주 중 · 2주차`
- `○○에게 맡긴 중`
- `○○를 관리 중`

#### 정책
- 클릭 불가
- 강조 금지
- 설명 문구 금지

정보는 안심용이다.

---

## 7. 마이크로 카피 가이드

| 상황 | 사용 금지 | 권장 |
|---|---|---|
| 실행 | 완료 | 했어 |
| 확인 | 승인 | 확인하기 |
| 성공 | 성공 | 확인됐어요 |
| 실패 | 실패 | 이번엔 못 했어 |

- 명령형 표현 금지
- 평가형 표현 금지
- 대화체, 관계 중심 표현 사용

---

## 8. 절대 포함하지 말 것

- ❌ 역할 토글
- ❌ 할 일 리스트
- ❌ 통계/그래프
- ❌ 연속 기록
- ❌ 성취율
- ❌ 랭킹/비교

홈은 조용한 관계 상태 카드다.

---

## 9. 이 화면의 한 문장 정의

> **OnMyBehalf의 홈은  
> 오늘 우리 사이에  
> 말이 필요한지를 보여주는 한 장의 카드다.**
