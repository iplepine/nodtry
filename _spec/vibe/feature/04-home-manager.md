# Screen 04 – 홈 (관리자 모드)
## Manager Home – Calm Oversight View

---

## 1. 화면 목적 (Why)

이 화면의 목적은 **감시가 아니다**.  
이 화면의 목적은 **조용한 책임 유지와 최소 개입**이다.

관리자가 느껴야 할 감정:
- “부담스럽지 않다”
- “필요한 것만 보면 된다”
- “내가 누군가를 돕고 있다”

👉 통제 ❌ / 압박 ❌ / 책임 유지 ⭕  

---

## 2. 행동 정책 (Behavior Policy)

### B1. 관리자 피로 최소화 원칙
- 관리 행동은 **최소 클릭**으로 끝나야 한다
- 판단은 빠르고 가볍게

### B2. 다중 관리 전제
- 관리자는 **여러 명을 동시에 관리할 수 있다**
- 한 번에 한 명만 집중해서 본다

### B3. 판단 중심 화면
- 이 화면은 “실행”을 유도하지 않는다
- 오직 **확인 / 승인 / 판단**만 제공한다

---

## 3. 사용자 상태 분기 (State Model)

### 상태 A. 관리 대상이 없음
- 메시지:
  - “아직 맡고 있는 사람이 없어요”
- CTA:
  - “초대 코드 만들기”

---

### 상태 B. 관리 대상은 있으나, 오늘 보고 없음
- 메시지:
  - “오늘은 모두 조용한 하루예요”
- CTA 없음

---

### 상태 C. 확인 대기 보고가 있음
- 상단 인박스 카드 표시
- 확인 액션 노출

---

## 4. 관리 대상 전환 UI (핵심)

### 4.1 관리 대상 선택 영역

#### 위치
- 화면 상단 고정

#### 형태
- Dropdown 또는 Chip Selector

#### 표시 정보
- 이름
- 오늘 상태 요약:
  - `미보고`
  - `보고 대기`
  - `확인 완료`

#### 기본 선택 규칙
- 가장 최근 활동한 대상
- 마지막 선택 대상 기억

---

### 4.2 관리 대상 변경 시 정책
- 즉시 컨텍스트 변경
- 이전 대상 상태 유지
- 데이터 손실 없음

---

## 5. UI 구성 요소

### 5.1 상단 요약 영역

#### 구성
- 선택된 관리 대상 이름
- 현재 관리 기간:
  - `4주 중 · 2주차`

#### 정책
- 성과 요약 ❌
- 퍼센트 ❌
- 비교 ❌

### 5.2 인박스 영역 (중앙)

#### 확인 대기 보고 카드

- 상태 텍스트:
  - “운동했다고 전달했어요”
- 액션 버튼:
  - `했다`
