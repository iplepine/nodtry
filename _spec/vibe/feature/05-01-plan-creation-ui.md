# OnMyBehalf – 계획 생성 화면 UI (유저 피드백 반영 최종안)
## Plan Creation UI – User Feedback Applied

---

## 0. 유저 사용성 피드백 요약

실제 사용자를 가정했을 때 발생 가능한 문제점은 다음과 같다.

1. "이걸 왜 지금 해야 하지?"라는 맥락 부족
2. 화면이 바뀔 때 내가 뭘 하고 있는지 잊음
3. 숫자 선택이 은근히 부담됨 (주 2/3/4회)
4. 요일 선택이 귀찮아서 이탈 가능
5. 마지막 요약 화면에서 '확정' 부담 발생
6. 관리자에게 보내는 순간 긴장감 상승

본 문서는 위 문제를 해결하도록 UI를 수정한다.

---

## 1. 전체 UX 구조 수정 요약

### 변경 핵심

- "계획 생성 플로우" → **"가벼운 제안 흐름"**
- 모든 화면에 **현재 단계 힌트** 제공
- 결정 → 제안 → 합의의 심리 단계를 명확히 분리

---

## 2. 공통 UI 개선 (모든 화면 적용)

### 2.1 진행 맥락 힌트 추가

#### 적용
- 화면 상단에 Progress Hint Bar 추가

표현 예:
- "제안 준비 중 · 1/3"
- "이 정도면 괜찮을지 정하는 중 · 2/3"
- "이렇게 제안할 거예요 · 3/3"

#### 효과
- 사용자가 '길을 잃었다'는 느낌 제거
- 중도 이탈 감소

---

### 2.2 결정 압박 완화 문구 추가

모든 CTA 주변에 다음 중 하나를 반드시 포함한다.

- "나중에 바꿔도 괜찮아요"
- "확정이 아니라 제안이에요"
- "부담되면 언제든 조정할 수 있어요"

---

## 3. Screen 1 – 행동 선택 화면 (개선)

### 유저 피드백
- "이걸 왜 고르는지 모르겠다"
- "계획 같아서 부담된다"

### 개선 적용

#### 상단 설명 추가
- SubTitle 변경:
  - 기존: 완벽하지 않아도 괜찮아요
  - 변경: "한 가지 약속만 정해볼까요?"

#### 카드 카피 수정
- 운동하기 → "몸을 좀 움직여볼까?"
- 아이와 약속 지키기 → "아이랑 한 가지 약속"
- 생활습관 하나 만들기 → "생활 리듬 하나"
- 직접 입력하기 → "직접 정해볼래요"

#### 효과
- '계획'이 아니라 '약속'으로 인식 전환

---

## 4. Screen 2 – 반복 빈도 설정 화면 (개선)

### 유저 피드백
- "주 3회가 맞나? 괜히 고민된다"
- "숫자 고르는 게 피곤하다"

### 개선 적용

#### 선택지 의미 재정의
- 주 2회 → "가볍게"
- 주 3회 → "적당히" (기본)
- 주 4회 → "조금 더"

숫자는 보조 정보로만 작게 표시

#### Helper Text 강화
- "완벽하게 안 지켜도 괜찮아요"
- "평균적으로 이 정도면 돼요"

#### 효과
- 숫자 결정 스트레스 감소
- 감정 기반 선택 유도

---

## 5. Screen 3 – 요일 선택 화면 (개선)

### 유저 피드백
- "요일 정하는 게 귀찮다"
- "넘기고 싶은데 괜히 미안하다"

### 개선 적용

#### 기본 상태 변경
- 요일 미선택 상태를 '정상 상태'로 명시

#### 문구 변경
- Title:
  - 기존: 요일을 정할까?
  - 변경: "요일은 안 정해도 돼요"

#### SkipAction 강조
- "그냥 그날 컨디션 보고 할게요" (기본 추천)

#### 효과
- 요일 선택 화면에서 이탈률 감소

---

## 6. Screen 4 – 계획 제안 요약 화면 (개선)

### 유저 피드백
- "이걸 누르면 확정되는 느낌"
- "상대에게 보내는 게 부담됨"

### 개선 적용

#### 카드 하단 설명 추가
- "이건 제안이에요. 상대가 보고 같이 정해요."

#### CTA 카피 변경
- 기존: 이렇게 제안할게요
- 변경: 이 정도면 어떨지 물어볼게요

#### 효과
- 심리적 '확정' 부담 제거
- 대화 시작 느낌 강화

---

## 7. 관리자 확인 화면 (개선)

### 유저 피드백
- "승인/거절은 너무 딱딱하다"
- "거절하면 미안하다"

### 개선 적용

#### 버튼 카피 변경
- 괜찮아 보여 → 이 정도면 괜찮아
- 다시 얘기해보자 → 조금만 조정해볼까

#### 상단 문구 추가
- "완벽하지 않아도 괜찮아요. 같이 맞춰가면 돼요."

#### 효과
- 거절에 대한 죄책감 감소
- 관계 긴장 완화

---

## 8. 전체 UX 변경 후 기대 효과

- 계획 생성 완료율 ↑
- 중간 이탈률 ↓
- 관리자 승인 거절률 ↓
- '감시 앱' 인식 제거
- 커플 대화 트리거 역할 강화

---

## 9. 최종 UX 정의 문장 (업데이트)

OnMyBehalf의 계획 화면은  
결정을 요구하는 화면이 아니라,  
"이 정도면 괜찮지?"라고  
상대에게 말을 건네는 화면이다.
